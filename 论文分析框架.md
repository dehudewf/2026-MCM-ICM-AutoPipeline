## 四、解题范式总结

### 通用解题流程

```
1. 问题分解
   ├─ 识别子问题
   ├─ 确定问题类型
   └─ 明确评价标准

2. 数据探索
   ├─ 描述性统计
   ├─ 相关性分析
   └─ 异常值处理

3. 模型选择
   ├─ 基准模型
   ├─ 改进模型
   └─ 集成策略

4. 模型求解
   ├─ 参数估计
   ├─ 算法实现
   └─ 结果输出

5. 验证与优化
   ├─ 交叉验证

6. 结果呈现
   ├─ 可视化
   ├─ 解释性分析
   └─ 政策建议
```

## 六、写作黄金法则

### 摘要（1页）
1. **问题重述**（2-3句）：简洁明了
2. **模型概述**（3-4句）：突出创新
3. **主要结果**（3-4句）：量化表述
4. **优势总结**（2-3句）：强调价值

### 正文结构
1. **Introduction**（1-1.5页）
   - 背景
   - 文献综述（简短）
   - 本文工作

2. **Problem Analysis**（1-2页）
   - 问题分解
   - 数据探索
   - 假设说明

3. **Model Development**（6-8页）
   - 模型1
   - 模型2
   - 模型集成

4. **Results & Discussion**（3-4页）
   - 结果展示
   - 敏感性分析
   - 模型验证

5. **Conclusions**（0.5-1页）
   - 主要发现
   - 优势与局限
   - 未来工作

### 图表原则
- **数量**：10-15个
- **类型**：流程图、结果图、对比图
- **质量**：清晰、专业、配色合理
- **说明**：每个图表都要在正文中引用和解释
