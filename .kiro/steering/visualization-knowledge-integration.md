## 🔧 系统集成方式

### 1. .cursorrules 快速命令

```yaml
# 可视化知识库调用
@knowledge:viz 检索{图表类型} 
  → A题: A题可视化知识库.csv (82种: 9大类别全覆盖)
```

### 2. mcm_ultimate_system.md 知识库映射

```yaml
knowledge_base:
  visualization:
    - path: "A题/知识库/A题可视化知识库.csv"
      usage: "A题图表类型选择 (82种可视化: 微分方程/优化算法/数值仿真/机器学习/统计分析/验证分析)"
```

### 3. Agent调用协议

```yaml
@strategist:
  - 审题阶段引用可视化类型指导模型选择
  - 创新发散时考虑图表表达创新点
  - A题需重点考虑：相图、Pareto前沿、元胞自动机演化图

@executor:
  - 实现阶段根据模型选择对应图表
  - 生成高质量可视化(300 DPI, grayscale-readable)
  - 每张图必须有informative caption
  - A题特色：相图、收敛曲线、敏感性龙卷风图、网格无关性图

@redcell:
  - 检查图表是否自解释
  - 验证图表与正文一致性
  - 确认所有图表被引用
  - A题必查：数值验证完整性、敏感性分析充分性、物理一致性验证
```

## ✅ A题可视化检查清单

### Phase 1: 审题阶段 (0-12h)
- [ ] 确定物理过程类型（扩散/传播/优化/动力学）
- [ ] @strategist 引用可视化知识库指导战略规划
- [ ] 确定核心图表类型（相图/收敛曲线/空间分布图）

### Phase 2: 建模阶段 (12-72h)
- [ ] @executor 根据模型选择对应可视化
- [ ] 生成数值验证图（网格收敛、误差分析、龙格库塔验证）
- [ ] 生成仿真结果图（浓度场、温度场、元胞自动机演化）
- [ ] 生成优化过程图（收敛曲线、Pareto前沿、种群分布）
- [ ] 生成微分方程图（相图、SIR/SEIR曲线、稳定性图）

### Phase 3: 写作阶段 (48-96h)
- [ ] 所有图表 300 DPI、grayscale-readable
- [ ] 每张图有informative caption
- [ ] 所有图表在正文被引用和讨论
- [ ] 敏感性分析图和稳定性图必须包含
- [ ] 论文呈现类表格完整（假设表、符号表、参数表）

### Phase 4: 检查阶段 (96-100h)
- [ ] @redcell 检查图表自解释性
- [ ] 验证图表编号连续
- [ ] 确认图表与文字一致
